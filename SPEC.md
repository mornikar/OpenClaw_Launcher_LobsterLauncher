---

## 一、产品定位

**龙虾启动器** 是一款专为 OpenClaw 用户打造的桌面端快捷管理工具，提供文件快速访问、API 使用监控、模型管理、终端集成、**本地服务启动控制**等一站式体验。

**核心用户**：OpenClaw 深度用户、开发者、AI 效率工作者

**核心价值**：减少日常操作路径，提升 OpenClaw 工作流效率，一键管理本地 AI 服务

- **路径动态检测**：首次启动时自动扫描可能的 OpenClaw 安装位置

- 卡片悬停发光效果

#### 4.2.2 字段说明

| 字段 | 说明 | 默认值 | 可编辑 |
|------|------|--------|--------|
| Gateway URL | Gateway 服务地址 | `http://localhost:18789` | ✅ |

#### 4.2.3 功能点

| 功能 | 说明 |
|------|------|
| 一键复制 | 点击复制按钮，复制到剪贴板 + 气泡提示"已复制" |
| 密码显示/隐藏 | 点击眼睛图标切换显示/隐藏 |
| 凭证刷新 | 从配置文件重新读取最新凭证 |
| 配置保存 | 编辑 Gateway URL 后保存到配置文件 |

#### 4.2.4 数据来源

```javascript
// 凭证信息来源
const credentials = {
    gatewayUrl: 'http://localhost:18789',      // 从配置读取
    token: fs.readFileSync('~/.openclaw/token', 'utf8'),  // 从文件读取
    password: ''                                // 用户手动输入保存
};
```

#### 4.2.5 UI 交互

- **复制成功**：按钮变绿 + 显示"已复制"1.5秒后恢复
- **Token 过长**：截断显示，点击复制完整值
- **密码输入**：密码模式显示，光标悬停显示实际值
- **配置读取**：启动时自动检测，异常显示"未检测到配置"

### 4.3.1 安装向导智能检测机制

> 点击"安装向导"按钮时，自动检测 OpenClaw 安装状态，提供最佳用户体验

**检测流程**：
```
用户点击"安装向导"
       ↓
检测 OpenClaw 目录是否存在 (~/.openclaw)
       ↓
    ┌───────┴───────┐
    ↓               ↓
  已安装           未安装
    ↓               ↓
显示确认对话框      显示安装确认对话框
"是否打开安装向导？"   "是否安装 OpenClaw？"
    ↓               ↓
用户点击"确定"      用户点击"确定"
    ↓               ↓
打开 Onboard 页面   显示安装进度弹窗
    |               ↓
    |         自动执行安装流程
    |               ↓
    |         完成后打开 Onboard
    └───────────────┘
              ↓
        安装向导页面
```

**安装流程（自动执行）**：
| 阶段 | 进度 | 说明 |
|------|:----:|------|
| preparing | 10% | 创建安装目录 (~/.openclaw) |
| installing | 30% | 安装 OpenClaw CLI (npm install -g openclaw) |
| initializing | 60% | 初始化配置 |
| starting | 80% | 启动 Gateway 后台服务 |
| completed | 100% | 安装完成，打开安装向导 |

**Onboard URL**：`http://localhost:18789/onboard/chat?session=agent%3Amain%3Amain`

### 4.4 服务管理（核心功能重构）

#### 4.4.1 默认服务（OpenClaw Gateway）

| 项目 | 说明 |
|------|------|
| 名称 | OpenClaw Gateway |
| 图标 | 🦞 龙虾图标 |
| 类型 | 系统核心服务（不可删除） |
| 状态检测 | 进程名/端口检测 |
| 右键菜单 | 仅"停止"选项 |
| 警告提示 | "Gateway 必须保持运行" |

**行为**：
- 启动时自动检测 Gateway 状态
- 停止前弹出确认对话框
- 意外关闭后自动重启提醒

#### 4.4.2 本地模型服务（可扩展）

| 示例服务 | 说明 |
|---------|------|
| LM Studio | 本地 LLM 运行工具 |
| Ollama | 本地模型管理 |
| Ollama WebUI | Ollama 网页界面 |
| Text Generation WebUI | 文字生成界面 |
| 自定义 | 用户添加的任意 exe |

**功能**：
- 添加本地 exe 文件
- 自动提取 exe 图标
- 启动状态实时监测（进程/端口）
- 单个启动 / 停止
- 启动延迟设置（避免资源冲突）

#### 4.4.3 右下角全局控制

| 按钮 | 功能 |
|------|------|
| ⚡ 全部启动 | 按顺序启动所有"未运行"的服务 |
| ⏹ 全部停止 | 关闭所有"运行中"的服务（除 Gateway） |

**交互细节**：
- 启动间隔可配置（默认 1000ms）
- 启动过程显示进度
- 失败服务红色高亮 + 重试按钮

## 十五、OpenClaw 路径动态检测方案

### 15.1 检测位置（按优先级）

```
1. 环境变量 %OPENCLAW_HOME%
2. %USERPROFILE%\.openclaw
3. %APPDATA%\openclaw
4. %LOCALAPPDATA%\openclaw
5. 常见安装目录:
   - C:\Program Files\OpenClaw
   - D:\OpenClaw
6. 注册表 (Windows)
7. 最近使用目录 (扫描快捷方式)
```

### 15.2 检测结果存储

首次检测成功后，结果缓存到 `services.json`，后续启动直接读取。
